# Бабусині іграшки

Статичний сайт-вітрина м’яких іграшок ручної роботи. Побудований на [Astro](https://astro.build),
без серверної частини та форм — замовлення оформлюються через месенджери.

## Можливості

- Каталог іграшок із колекцій вмісту Astro (файли Markdown).
- Сторінка кожної іграшки з оглядачем обертання на 360°.
- Автоматичне стискання фото у формат WebP.
- Адаптивна верстка «спершу мобільний» (контрольні точки 640 px і 1024 px).
- SEO-теги та Open Graph на кожній сторінці, карта сайту.
- Кнопка «Замовити» з готовим повідомленням у Telegram.

## Вимоги

- Node.js версії 18.20, 20.3 або новішої (рекомендовано 20+).
- npm (постачається разом із Node.js).

## Локальний запуск

```bash
npm install      # встановити залежності (один раз)
npm run dev      # запустити сайт для розробки
```

Сайт відкриється за адресою `http://localhost:4321`.

Інші команди:

```bash
npm run build    # зібрати сайт у теку dist/
npm run preview  # переглянути зібраний сайт локально
```

## Налаштування контактів

Усі контакти та назва сайту зібрані в одному файлі — `src/config.ts`.
Відредагуйте значення перед розгортанням:

- `name`, `tagline`, `description` — назва, підпис і опис для SEO;
- `phone`, `phoneDisplay` — телефон для посилання та для показу;
- `telegram` — ім’я користувача Telegram **без** символу `@`;
- `viber` — номер Viber у міжнародному форматі без `+`;
- `instagram` — ім’я користувача Instagram без `@`.

Також замініть адресу сайту в `astro.config.mjs` (поле `site`) — вона потрібна
для карти сайту та абсолютних посилань Open Graph.

## Як додати нову іграшку

1. **Створіть фото.** Покладіть головне фото та додаткові знімки в теку
   `src/assets/toys/`. Формат — PNG або JPG (стискання у WebP відбувається
   автоматично під час збірки).

2. **Підготуйте кадри обертання** (за бажанням). Дивіться розділ нижче.
   Готові кадри мають опинитися в `public/spin/<slug>/`.

3. **Створіть запис.** Додайте файл `src/content/igrashky/<slug>.md`.
   Скопіюйте за зразком наявний запис і заповніть поля:

   ```markdown
   ---
   title: Назва іграшки
   slug: nazva-igrashky              # латиницею, збігається з іменем файлу
   price: 800                        # ціна у гривнях (число)
   size: 26 см
   materials: плюш, холлофайбер
   status: available                 # available | made-to-order | sold
   cover: ../../assets/toys/nazva-igrashky-cover.png
   spinFrames: 24                    # кількість кадрів обертання
   spinDir: nazva-igrashky           # тека в public/spin/
   gallery:
     - ../../assets/toys/nazva-igrashky-1.png
     - ../../assets/toys/nazva-igrashky-2.png
   ---

   Опис іграшки. Звичайний текст Markdown.
   ```

   Пояснення полів:
   - `status`: `available` — в наявності, `made-to-order` — під замовлення,
     `sold` — продано (картка автоматично приглушується на 55%, зникає ефект збільшення/нахилу при наведенні).
     *Важливо:* Для статусів `sold` та `made-to-order` на сторінці товару автоматично виводиться відповідне інформаційне повідомлення (про продаж або в'язання під замовлення). Додавати цей текст вручну в опис більше не потрібно.
   - `cover` — головне вертикальне фото (рекомендоване співвідношення сторін **3:4**) для картки й сторінки;
   - `gallery` — додаткові фото (мініатюри під оглядачем обертання). Поле є необов'язковим (можна залишити порожнім: `gallery: []`);
   - `spinDir` — назва теки з кадрами всередині `public/spin/`. Якщо обертання 360° немає, просто видаліть це поле (або не вказуйте значення).

4. **Перевірте.** Запустіть `npm run dev` — нова іграшка з’явиться в каталозі,
   а її сторінка буде доступна за адресою `/igrashky/<slug>/`.

## Як підготувати кадри обертання

Оглядач обертання читає кадри `frame-01.webp`, `frame-02.webp` … з теки
`public/spin/<slug>/`. Щоб підготувати їх з набору знятих по колу фотографій,
скористайтеся допоміжним скриптом:

```bash
node scripts/prepare-spin.mjs <тека-з-оригіналами> public/spin/<slug>
```

Приклад:

```bash
node scripts/prepare-spin.mjs ~/foto/vedmedyk public/spin/vedmedyk-tymko
```

Скрипт:

1. читає всі зображення з теки (у порядку імен файлів);
2. зменшує кожне до 1200 px по довшій стороні;
3. конвертує у WebP якістю 80;
4. перейменовує послідовно у `frame-01.webp`, `frame-02.webp`, …

Після завершення скрипт підкаже, яке значення вказати в полі `spinFrames`.

> Порада: знімайте іграшку на поворотному столику з однакової відстані,
> 24–36 кадрів за повний оберт дають плавне обертання.

## Особливості кешування в Astro 5 (Content Layer)

Astro 5 кешує дані контент-колекцій у тимчасовій теці `.astro/`. Через це при видаленні файлів іграшок або значній зміні шляхів до фотографій ви можете зіткнутися з помилкою збірки (наприклад, `ImageNotFound` для файлів, які вже були видалені).

**Вирішення:**
Якщо dev-сервер видає помилку про відсутність старих картинок, просто зупиніть процес у терміналі (`Ctrl + C`) та запустіть його знову:

```bash
npm run dev
```

Сервер оновить кеш, і збірка пройде успішно.

## Розгортання на Cloudflare Pages

1. Завантажте проєкт у репозиторій GitHub (або GitLab).
2. У панелі Cloudflare відкрийте **Workers & Pages → Create → Pages**
   і під’єднайте репозиторій.
3. Укажіть параметри збірки:
   - **Framework preset:** Astro;
   - **Build command:** `npm run build`;
   - **Build output directory:** `dist`.
4. Натисніть **Save and Deploy**. Cloudflare збере сайт і надасть адресу
   виду `https://<проєкт>.pages.dev`.
5. Впишіть цю адресу в поле `site` у `astro.config.mjs` і зробіть новий коміт,
   щоб карта сайту та Open Graph використовували правильний домен.

Кожен наступний коміт у головну гілку автоматично оновлюватиме сайт.

## Структура проєкту

```
babusyni-igrashky/
├── astro.config.mjs           # налаштування Astro та карти сайту
├── src/
│   ├── config.ts              # назва сайту й контакти
│   ├── content.config.ts      # схема колекції іграшок
│   ├── content/igrashky/      # записи іграшок (Markdown)
│   ├── assets/toys/           # фото іграшок (стискаються у WebP)
│   ├── components/            # компоненти сторінок
│   ├── layouts/               # базовий макет із SEO
│   ├── pages/                 # головна та сторінка іграшки
│   └── styles/global.css      # змінні палітри й базові стилі
├── public/
│   ├── spin/<slug>/           # кадри обертання (WebP)
│   ├── favicon.svg
│   └── og-default.png
└── scripts/prepare-spin.mjs   # підготовка кадрів обертання
```
