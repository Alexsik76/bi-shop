# Бабусині іграшки

Статичний сайт-вітрина м'яких іграшок ручної роботи. Побудований на [Astro](https://astro.build) та розгорнутий на **Cloudflare Pages**. Замовлення оформлюються через месенджери — кошика та форм оформлення немає.

## Архітектура

Проект побудований за гібридною edge-схемою:

- **Статичний HTML** — Astro генерує статичну оболонку при збірці (`dist/`), включаючи головну сторінку та універсальний шаблон деталей товару `/igrashky/_toy/`.
- **Cloudflare KV** (`TOYS_KV`) — є єдиним джерелом правди для всіх товарів. Зберігає текстові поля кожної іграшки: назву, ціну, розмір, матеріали, статус та опис. Ключ запису має вигляд `toy:<id>`.
- **Cloudflare R2** — зберігає зображення іграшок (обкладинку, галерею та кадри обертання 360°), адреси яких формуються динамічно за фіксованими правилами на основі ID іграшки та лічильників з KV.
- **Edge Middleware** (`functions/_middleware.ts` та `functions/_helpers.ts`) — перехоплює запити, списки та деталі іграшок. Зчитує дані з KV, динамічно генерує блоки галереї та обертання 360°, інжектує дані в HTML-код за допомогою `HTMLRewriter` та повертає готовий результат.
- **Панель керування** (`/admin`, `functions/admin/`) — захищена за допомогою Cloudflare Access. Дозволяє контент-менеджеру редагувати записи KV безпосередньо в базі даних без деплою коду.

Іграшка відображається у каталозі лише якщо в KV є запис із непорожньою назвою, ціною > 0 та дійсним статусом (перевірка `isVisible`).

## Можливості

- Динамічний каталог іграшок, що рендериться на рівні edge з бази даних Cloudflare KV.
- Захищена панель керування `/admin` для редагування записів KV без деплою.
- Сторінка кожної іграшки з оглядачем обертання на 360°.
- Трирівнева адаптивна система зображень (480w, 960w, 1600w WebP) з R2 для оптимального завантаження на різних пристроях.
- Адаптивна верстка «спершу мобільний» (контрольні точки 640 px і 1024 px).
- SEO-теги, Open Graph та JSON-LD розмітка, яка автоматично оновлюється на edge.
- Кнопка «Замовити» з готовим повідомленням у Telegram, що підставляє назву іграшки.

## Вимоги

- Node.js версії 18.20, 20.3 або новішої (рекомендовано 20+).
- npm (постачається разом із Node.js).
- [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/) для локального запуску з функціями та KV.

## Локальний запуск

Оскільки сайт повністю базується на Cloudflare KV та Pages Functions, локальний запуск рекомендовано виконувати за допомогою Wrangler:

```bash
npm install      # встановити залежності (один раз)
npm run build    # зібрати статичний сайт у теку dist/
npx wrangler pages dev dist --kv TOYS_KV --binding ADMIN_LOCAL_DEV=1
```

Сайт із функціями відкриється за адресою `http://127.0.0.1:8788`.

> [!NOTE]
> Локальний Wrangler автоматично створить локальну базу даних KV. Для першого заповнення бази даних відкрийте панель `/admin` локально.
> **Ніколи не додавайте `ADMIN_LOCAL_DEV` у налаштування розгорнутого проєкту на Cloudflare Pages.**

## Налаштування контактів

Усі контакти та назва сайту зібрані в одному файлі — `src/config.ts`.
Відредагуйте значення перед розгортанням:

- `name`, `tagline`, `description` — назва, підпис і опис для SEO;
- `phone`, `phoneDisplay` — телефон для посилання та для показу;
- `telegram` — ім'я користувача Telegram **без** символу `@`;
- `viber` — номер Viber у міжнародному форматі без `+`;
- `instagram` — ім'я користувача Instagram без `@`;
- `orderMessage` — шаблон повідомлення при натисканні «Замовити» (`{title}` замінюється назвою іграшки);
- `statusNotes` — тексти приміток під описом для статусів `sold` та `made-to-order`.
- `r2Url` — базова адреса доставки зображень з Cloudflare R2.

Також замініть адресу сайту в `astro.config.mjs` (поле `site`) — вона потрібна для карти сайту та Open Graph.

## Додавання нових іграшок та зображень

Іграшки та зображення створюються зовнішнім сервісом **bi-studio** (який завантажує фотографії в Cloudflare R2 та ініціалізує записи в KV). 

Схема іменування зображень в R2 (під доменним ім'ям `site.r2Url`):
- Головне фото (обкладинка): `<id>/cover-1600.webp`, `<id>/cover-960.webp`, `<id>/cover-480.webp`.
- Додаткові фото: `<id>/gallery-<n>-1600.webp`, `-960.webp`, `-480.webp` (для `n` від 1 до `galleryCount`).
- Кадри обертання 360°: `<id>/spin/frame-<NN>.webp` (двозначний номер кадру, для `n` від 01 до `spinCount`).

Після того, як зображення завантажені, відкрийте `/admin` для заповнення текстових даних та публікації іграшки.

## Панель керування `/admin`

Захищена сторінка для редагування мутабельних полів іграшок безпосередньо в Cloudflare KV.

### Захист

Панель захищена **Cloudflare Access** (JWT-перевірка заголовка `Cf-Access-Jwt-Assertion`). Без дійсного токена повертається `403 Forbidden`.

Необхідні змінні оточення в налаштуваннях Cloudflare Pages (Production та Preview):

| Змінна | Опис |
|---|---|
| `CF_ACCESS_TEAM_DOMAIN` | Домен команди Zero Trust, наприклад `myteam.cloudflareaccess.com` |
| `CF_ACCESS_AUD` | Audience Tag із налаштувань Access Application |

### Формат запису в KV

Ключ: `toy:<id>`. Значення — JSON:

```json
{
  "title": "Назва іграшки",
  "price": 1250,
  "size": "28 см",
  "materials": "велюр, холлофайбер",
  "status": "available",
  "description": "Опис. Порожній рядок або \\n розділяє абзаци.",
  "galleryCount": 3,
  "spinCount": 36,
  "updatedAt": "2026-07-08T16:00:00.000Z"
}
```

`status` — одне з: `available`, `made-to-order`, `sold`. `galleryCount` та `spinCount` вказують на кількість фото в R2 та використовуються для динамічної генерації посилань (ці поля доступні в адмінці тільки для читання).

## Розгортання на Cloudflare Pages

1. Завантажте проєкт у репозиторій GitHub (або GitLab).
2. У панелі Cloudflare відкрийте **Workers & Pages → Create → Pages** і під'єднайте репозиторій.
3. Укажіть параметри збірки:
   - **Framework preset:** Astro;
   - **Build command:** `npm run build`;
   - **Build output directory:** `dist`.
4. У налаштуваннях проекту → **KV namespace bindings** прив'яжіть namespace `TOYS_KV`.
5. Додайте змінні оточення `CF_ACCESS_TEAM_DOMAIN` та `CF_ACCESS_AUD`.
6. Натисніть **Save and Deploy**.
7. Впишіть отриману адресу в поле `site` у `astro.config.mjs` і зробіть новий коміт.

```
├── astro.config.mjs           # налаштування Astro, маршрутизації та карти сайту
├── wrangler.toml              # конфіг Cloudflare Pages: KV binding
├── package.json               # залежності та скрипти збірки
├── functions/                 # Cloudflare Pages Functions (edge)
│   ├── _middleware.ts         # маршрутизація edge-запитів та реврайт HTML
│   ├── _helpers.ts            # помічники з генерації HTML та розмітки
│   └── admin/                 # панель керування /admin
│       ├── index.ts           # перелік іграшок з KV, валідація та збереження
│       ├── auth.ts            # перевірка Cloudflare Access JWT
│       └── template.ts        # HTML/CSS шаблон панелі
├── src/
│   ├── config.ts              # назва сайту, контакти, шаблон замовлення, r2Url
│   ├── assets/                # спільні графічні ресурси сайту
│   ├── components/            # Astro компоненти (Header, Footer, Hero, HowToOrder, OrderButton)
│   ├── layouts/               # базовий макет із SEO
│   ├── pages/                 # сторінки сайту
│   │   ├── index.astro        # головна сторінка (оболонка каталогу)
│   │   ├── about.astro        # сторінка «Про майстриню»
│   │   ├── 404.astro          # сторінка помилки
│   │   └── igrashky/
│   │       └── _toy/
│   │           └── index.astro # єдиний загальний шаблон деталей іграшки (shell)
│   └── styles/global.css      # змінні палітри й базові стилі
├── public/
│   ├── favicon.ico
│   ├── favicon.svg
│   ├── apple-touch-icon.png
│   ├── og-default.png
│   └── robots.txt
└── scripts/                   # локальний інструментарій розробника (legacy / reference)
```
