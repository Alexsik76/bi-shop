# Бабусині іграшки

Статичний сайт-вітрина м'яких іграшок ручної роботи. Побудований на [Astro](https://astro.build), розгорнутий на **Cloudflare Pages**. Замовлення оформлюються через месенджери — кошика та форм оформлення немає.

## Архітектура

Проект побудований за гібридною схемою:

- **Статичний HTML** генерується Astro при збірці (`dist/`) і містить заглушки для динамічних даних.
- **Cloudflare KV** (`TOYS_KV`) зберігає мутабельні поля кожної іграшки: назву, ціну, розмір, матеріали, статус і опис.
- **Edge Middleware** (`functions/_middleware.ts`) перехоплює запити та підставляє дані з KV у статичні сторінки через `HTMLRewriter` — без серверного рендерингу.
- **Панель керування** (`/admin`, `functions/admin/`) захищена Cloudflare Access і дозволяє оператору редагувати записи KV без деплою.

Іграшка відображається у каталозі лише якщо в KV є запис із непорожньою назвою, ціною > 0 та дійсним статусом.

## Можливості

- Каталог іграшок із колекцій вмісту Astro (файли Markdown — лише зображення).
- Дані іграшок (назва, ціна, статус тощо) зберігаються в Cloudflare KV та інжектуються на сторінки на рівні edge.
- Захищена панель керування `/admin` для редагування записів KV без деплою.
- Сторінка кожної іграшки з оглядачем обертання на 360°.
- Автоматичне стискання фото у формат WebP.
- Адаптивна верстка «спершу мобільний» (контрольні точки 640 px і 1024 px).
- SEO-теги та Open Graph на кожній сторінці, карта сайту.
- Кнопка «Замовити» з готовим повідомленням у Telegram.

## Вимоги

- Node.js версії 18.20, 20.3 або новішої (рекомендовано 20+).
- npm (постачається разом із Node.js).
- [Wrangler CLI](https://developers.cloudflare.com/workers/wrangler/) для локального запуску з функціями та KV.

## Локальний запуск

### Тільки сайт (без функцій та KV)

```bash
npm install      # встановити залежності (один раз)
npm run dev      # запустити сайт для розробки
```

Сайт відкриється за адресою `http://localhost:4321`.

### З Cloudflare Functions та KV (рекомендовано)

```bash
npm run build                                          # зібрати статичний сайт
npx wrangler pages dev dist --kv TOYS_KV               # запустити з Pages Functions та KV
```

Сайт із функціями відкриється за адресою `http://127.0.0.1:8788`.

> [!NOTE]
> Щоб панель `/admin` була доступна локально, додайте файл `.dev.vars` у корінь проекту:
> ```
> ADMIN_LOCAL_DEV=1
> ```
> **Ніколи не додавайте `ADMIN_LOCAL_DEV` у налаштування розгорнутого проекту на Cloudflare Pages.**

Інші команди:

```bash
npm run build    # зібрати сайт у теку dist/
npm run preview  # переглянути зібраний сайт локально (без функцій)
```

## Налаштування контактів

Усі контакти та назва сайту зібрані в одному файлі — `src/config.ts`.
Відредагуйте значення перед розгортанням:

- `name`, `tagline`, `description` — назва, підпис і опис для SEO;
- `phone`, `phoneDisplay` — телефон для посилання та для показу;
- `telegram` — ім'я користувача Telegram **без** символу `@`;
- `viber` — номер Viber у міжнародному форматі без `+`;
- `instagram` — ім'я користувача Instagram без `@`;
- `orderMessage` — шаблон повідомлення при натисканні «Замовити» (`{title}` замінюється назвою іграшки);
- `statusNotes` — тексти приміток під описом для статусів `sold` та `made-to-order`.

Також замініть адресу сайту в `astro.config.mjs` (поле `site`) — вона потрібна
для карти сайту та абсолютних посилань Open Graph.

## Як додати нову іграшку

Рекомендований спосіб — скористатися автоматичним скриптом `new-toy.mjs`.

> [!IMPORTANT]
> Скрипт створює **лише структуру папок та зображення**. Мутабельні дані (назву, ціну, статус, опис тощо) слід заповнити через **панель керування `/admin`** після розгортання або через Wrangler KV локально.

### Спосіб 1: Через веб-інтерфейс (рекомендовано)

Запустіть скрипт без параметрів у терміналі:

```bash
node scripts/new-toy.mjs
```

Скрипт автоматично запустить локальний веб-сервер та відкриє у браузері зручний веб-інтерфейс (GUI). У ньому ви зможете:
- Ввести ID іграшки (наприклад `vedmedyk-tymko`).
- Перевірити в реальному часі, чи не зайнятий уже цей ID.
- Завантажити головне фото (обкладинку) та додаткові фотографії для галереї за допомогою Drag-and-Drop.
- Вибрати спосіб створення 360° обертання: без обертання, з відеофайлу (`.mp4`, `.mov`) або з готової теки кадрів.
- Спостерігати за процесом обробки в реальному часі.
- Після завершення натиснути «Зупинити сервер 🔌» для зупинки процесу.

Скрипт автоматично оптимізує фотографії та створює структуру папок з `index.md`. Потім відкрийте `/admin` і заповніть назву, ціну, статус та опис.

### Спосіб 2: Через консольний headless-режим

```bash
node scripts/new-toy.mjs --headless <id> <тека-з-фото-чи-відео> [--frames N]
```

Параметри:
- `<id>` — унікальний ідентифікатор іграшки (наприклад `vedmedyk-tymko`).
- `<тека-з-фото-чи-відео>` — шлях до теки з оригінальними фотографіями (та опціонально відео обертання чи теки `spin/`).
- `--frames N` — кількість кадрів для вилучення з відео (за замовчуванням 36).

### Спосіб 3: Ручне створення

1. **Створіть теку іграшки.** Додайте нову теку в `src/content/igrashky/<id>/`.

2. **Створіть маніфест.** Додайте файл `index.md` із посиланнями на зображення:

   ```markdown
   ---
   cover: ./cover.webp
   gallery:
     - ./gallery-1.webp
     - ./gallery-2.webp
   ---
   ```

   > [!WARNING]
   > **Не додавайте** поля `title`, `price`, `size`, `materials`, `status` та `description` в `index.md` — вони зберігаються виключно в Cloudflare KV і редагуються через `/admin`. Тіло Markdown (текст після `---`) також ігнорується.

3. **Підготуйте кадри обертання** (за бажанням). Готові кадри мають опинитися в підтеці `spin/` поряд з `index.md`. Кількість кадрів визначається автоматично. Вкажіть поле `spinDir: ./spin` у `index.md`.

4. **Заповніть дані в KV.** Після розгортання (або локально через Wrangler) відкрийте `/admin` і введіть назву, ціну, статус та опис іграшки.

5. **Перевірте.** Запустіть `npx wrangler pages dev dist --kv TOYS_KV` — іграшка з'явиться в каталозі після заповнення даних у `/admin`.

## Як підготувати фотографії

```bash
node scripts/prepare-photos.mjs <шлях-до-оригіналів-чи-файлу> <тека-іграшки>
```

Приклад:
```bash
node scripts/prepare-photos.mjs ~/foto/vedmedyk-tymko src/content/igrashky/vedmedyk-tymko
```

Скрипт:
1. Зчитує одне або декілька зображень із вказаного шляху (підтримуються `.jpg`, `.jpeg`, `.png`, `.webp`, `.heic` тощо).
2. Сортує їх за алфавітом.
3. Перше фото (або файл, що містить `cover` у назві) стає головним фото `cover.webp`. Решта стають `gallery-1.webp`, `gallery-2.webp` тощо.
4. Стискає до макс. 2400 px по довшій стороні та конвертує у WebP з якістю 90%.
5. Перед записом очищує старі версії фотографій у цільовій теці.

## Як підготувати кадри обертання

```bash
node scripts/prepare-spin.mjs <оригінали-чи-відео> [тека-призначення] [--frames N]
```

Параметри:
- `<оригінали-чи-відео>` — шлях до теки з оригіналами або до відеофайлу (`.mp4`, `.mov`).
- `[тека-призначення]` — опціональний шлях. Якщо не вказано, використовується `src/content/igrashky/<назва>/spin/`.
- `--frames N` — кількість кадрів з відео (за замовчуванням `36`).

> [!NOTE]
> Для роботи з відеофайлами потрібні `ffmpeg` та `ffprobe` у системному PATH.

Скрипт:
1. Якщо на вхід подано відео — витягує `N` кадрів через `ffmpeg` у тимчасову теку.
2. Читає всі зображення у порядку імен файлів.
3. Зменшує кожне до 1200 px по довшій стороні.
4. Конвертує у WebP якістю 80.
5. Перейменовує послідовно у `frame-01.webp`, `frame-02.webp`, …

> Порада: знімайте іграшку на поворотному столику з однакової відстані,
> 24–36 кадрів за повний оберт дають плавне обертання.

## Особливості кешування в Astro 5 (Content Layer)

Astro 5 кешує дані контент-колекцій у тимчасовій теці `.astro/`. При видаленні файлів або значній зміні шляхів до фотографій може виникати помилка збірки (`ImageNotFound`).

**Вирішення:** зупиніть процес (`Ctrl + C`) і запустіть знову:

```bash
npm run dev
```

## Панель керування `/admin`

Захищена сторінка для редагування мутабельних полів іграшок безпосередньо в Cloudflare KV.

### Захист

Панель захищена **Cloudflare Access** (JWT-перевірка заголовка `Cf-Access-Jwt-Assertion`). Без дійсного токена повертається `403 Forbidden`.

Необхідні змінні оточення в налаштуваннях Cloudflare Pages (Production та Preview):

| Змінна | Опис |
|---|---|
| `CF_ACCESS_TEAM_DOMAIN` | Домен команди Zero Trust, наприклад `myteam.cloudflareaccess.com` |
| `CF_ACCESS_AUD` | Audience Tag із налаштувань Access Application |

### Налаштування Cloudflare Access (ручні кроки)

1. У [Zero Trust Dashboard](https://one.dash.cloudflare.com/) → **Access → Applications → Add**.
2. Тип: **Self-Hosted**. Укажіть домен і шлях `/admin`.
3. Додайте політику доступу (наприклад, Email OTP для адреси оператора).
4. Скопіюйте **Audience Tag** і додайте як `CF_ACCESS_AUD` у налаштуваннях Pages.

### Формат запису в KV

Ключ: `toy:<id>`. Значення — JSON:

```json
{
  "title": "Назва іграшки",
  "price": 1250,
  "size": "28 см",
  "materials": "велюр, холлофайбер",
  "status": "available",
  "description": "Опис. Порожній рядок або \\n розділяє абзаци.",
  "updatedAt": "2026-07-08T16:00:00.000Z"
}
```

`status` — одне з: `available`, `made-to-order`, `sold`.

## Розгортання на Cloudflare Pages

1. Завантажте проєкт у репозиторій GitHub (або GitLab).
2. У панелі Cloudflare відкрийте **Workers & Pages → Create → Pages** і під'єднайте репозиторій.
3. Укажіть параметри збірки:
   - **Framework preset:** Astro;
   - **Build command:** `npm run build`;
   - **Build output directory:** `dist`.
4. У налаштуваннях проекту → **KV namespace bindings** прив'яжіть namespace `TOYS_KV`.
5. Додайте змінні оточення `CF_ACCESS_TEAM_DOMAIN` та `CF_ACCESS_AUD`.
6. Натисніть **Save and Deploy**.
7. Впишіть отриману адресу в поле `site` у `astro.config.mjs` і зробіть новий коміт.

Кожен наступний коміт у головну гілку автоматично оновлюватиме сайт.

## Структура проєкту

```
babusyni-igrashky/
├── astro.config.mjs           # налаштування Astro та карти сайту
├── wrangler.toml              # конфіг Cloudflare Pages: KV binding
├── .dev.vars                  # локальні змінні (не комітити! — у .gitignore)
├── functions/                 # Cloudflare Pages Functions (edge)
│   ├── _middleware.ts         # інжекція KV-даних у статичні сторінки
│   └── admin/                 # панель керування /admin
│       ├── index.ts           # маршрутизація та валідація
│       ├── auth.ts            # перевірка Cloudflare Access JWT
│       └── template.ts        # HTML/CSS шаблон панелі
├── src/
│   ├── config.ts              # назва сайту, контакти, шаблон замовлення
│   ├── content.config.ts      # схема колекції іграшок (лише зображення)
│   ├── content/igrashky/      # колекція іграшок
│   │   └── <id>/              # самодостатня тека іграшки (id = slug)
│   │       ├── index.md       # лише зображення: cover, gallery, spinDir
│   │       ├── cover.webp     # головне фото
│   │       ├── gallery-1.webp # додаткові фото (якщо є)
│   │       └── spin/          # кадри обертання (якщо є)
│   │           ├── frame-01.webp
│   │           └── ...
│   ├── assets/                # спільні графічні ресурси сайту
│   ├── components/            # компоненти сторінок
│   ├── layouts/               # базовий макет із SEO
│   ├── lib/                   # допоміжні утиліти (status.ts тощо)
│   ├── pages/                 # сторінки сайту
│   │   ├── index.astro        # головна сторінка (каталог)
│   │   ├── about.astro        # сторінка «Про майстриню»
│   │   ├── 404.astro          # сторінка помилки
│   │   ├── toys.json.ts       # маніфест іграшок для edge middleware
│   │   └── igrashky/
│   │       └── [slug].astro   # сторінка окремої іграшки
│   └── styles/global.css      # змінні палітри й базові стилі
├── public/
│   ├── favicon.svg
│   ├── apple-touch-icon.png
│   ├── og-default.png
│   └── robots.txt
└── scripts/                   # допоміжні скрипти автоматизації
    ├── new-toy.mjs            # головна точка входу (GUI або CLI)
    ├── pipeline.mjs           # конвеєр обробки фото та формування маніфесту
    ├── prepare-photos.mjs     # обробка та оптимізація фотографій галереї
    ├── prepare-spin.mjs       # підготовка кадрів обертання
    ├── server.mjs             # локальний HTTP-сервер для Web GUI
    └── gui/                   # файли веб-інтерфейсу (HTML, CSS)
```
