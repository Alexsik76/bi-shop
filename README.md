# Бабусині іграшки

Статичний сайт-вітрина м’яких іграшок ручної роботи. Побудований на [Astro](https://astro.build),
без серверної частини та форм — замовлення оформлюються через месенджери.

## Можливості

- Каталог іграшок із колекцій вмісту Astro (файли Markdown).
- Сторінка кожної іграшки з оглядачем обертання на 360°.
- Автоматичне стискання фото у формат WebP.
- Адаптивна верстка «спершу мобільний» (контрольні точки 640 px і 1024 px).
- SEO-теги та Open Graph на кожній сторінці, карта сайту.
- Кнопка «Замовити» з готовим повідомленням у Telegram.

## Вимоги

- Node.js версії 18.20, 20.3 або новішої (рекомендовано 20+).
- npm (постачається разом із Node.js).

## Локальний запуск

```bash
npm install      # встановити залежності (один раз)
npm run dev      # запустити сайт для розробки
```

Сайт відкриється за адресою `http://localhost:4321`.

Інші команди:

```bash
npm run build    # зібрати сайт у теку dist/
npm run preview  # переглянути зібраний сайт локально
```

## Налаштування контактів

Усі контакти та назва сайту зібрані в одному файлі — `src/config.ts`.
Відредагуйте значення перед розгортанням:

- `name`, `tagline`, `description` — назва, підпис і опис для SEO;
- `phone`, `phoneDisplay` — телефон для посилання та для показу;
- `telegram` — ім’я користувача Telegram **без** символу `@`;
- `viber` — номер Viber у міжнародному форматі без `+`;
- `instagram` — ім’я користувача Instagram без `@`.

Також замініть адресу сайту в `astro.config.mjs` (поле `site`) — вона потрібна
для карти сайту та абсолютних посилань Open Graph.

## Як додати нову іграшку

Рекомендований спосіб — скористатися автоматичним скриптом `new-toy.mjs`.

### Спосіб 1: Через веб-інтерфейс (рекомендовано)

Запустіть скрипт без параметрів у терміналі:

```bash
node scripts/new-toy.mjs
```

Скрипт автоматично запустить локальний веб-сервер та відкриє у браузері зручний веб-інтерфейс (GUI). У ньому ви зможете:
- Ввести ID іграшки (slug, наприклад `vedmedyk-tymko`).
- Перевірити в реальному часі, чи не зайнятий уже цей ID іграшки.
- Завантажити головне фото (обкладинку) та додаткові фотографії для галереї за допомогою Drag-and-Drop.
- Вибрати спосіб створення 360° обертання: без обертання, з відеофайлу (`.mp4`, `.mov`) або з готової теки кадрів.
- Спостерігати за процесом обробки в реальному часі (детальний вивід консолі транслюється прямо на сторінку).
- Після завершення натиснути кнопку «Зупинити сервер 🔌» у верхньому правому куті для зупинки процесу Node.js.

Скрипт автоматично оптимізує фотографії, створює потрібну структуру папок і генерує початковий маніфест `index.md`. Після цього вам залишається лише відредагувати `index.md` та дописати текстові поля (назву, ціну, розмір, матеріали, статус та опис).

### Спосіб 2: Через консольний headless-режим

Якщо ви бажаєте додати іграшку з консолі без запуску браузера, використовуйте прапорець `--headless`:

```bash
node scripts/new-toy.mjs --headless <id> <тека-з-фото-чи-відео> [--frames N]
```

Параметри:
- `<id>` — унікальний ідентифікатор іграшки (slug, наприклад `vedmedyk-tymko`).
- `<тека-з-фото-чи-відео>` — шлях до теки з оригінальними фотографіями іграшки (та опціонально відео обертання чи теки `spin/`).
- `--frames N` — кількість кадрів для вилучення, якщо виявлено відео обертання (за замовчуванням 36).

Скрипт самостійно знайде файл обкладинки (шукає слово `cover` у назві, інакше бере перше зображення), сформує галерею з решти зображень, обробить відео чи окрему теку обертання та автоматично запише маніфест `index.md`.

### Спосіб 3: Ручне створення (альтернатива)

1. **Створіть теку іграшки.** Додайте нову теку в `src/content/igrashky/<slug>/` (наприклад, `src/content/igrashky/vedmedyk/`).

2. **Створіть запис.** Додайте файл `index.md` у створену тему (`src/content/igrashky/<slug>/index.md`). Заповніть поля:

   ```markdown
   ---
   title: Назва іграшки
   price: 800                        # ціна у гривнях (число)
   size: 26 см
   materials: плюш, холлофайбер
   status: available                 # available | made-to-order | sold
   cover: ./cover.webp                # відносний шлях до головного фото
   gallery:
     - ./gallery-1.webp               # відносний шлях до додаткового фото
     - ./gallery-2.webp
   ---

   Опис іграшки. Звичайний текст Markdown.
   ```

   Пояснення полів:
   - `status`: `available` — в наявності, `made-to-order` — під замовлення,
     `sold` — продано (картка автоматично приглушується на 55%, зникає ефект збільшення/нахилу при наведенні).
     *Важливо:* Для статусів `sold` та `made-to-order` на сторінці товару автоматично виводиться відповідне інформаційне повідомлення (про продаж або в'язання під замовлення). Додавати цей текст вручну в опис більше не потрібно.
   - `cover` — головне вертикальне фото (рекомендоване співвідношення сторін **3:4**) для картки й сторінки (наприклад, `cover.webp` у тій же теці);
   - `gallery` — додаткові фото (мініатюри під оглядачем обертання). Поле є необов'язковим (можна залишити порожнім: `gallery: []`).

3. **Підготуйте кадри обертання** (за бажанням). Дивіться розділ нижче.
   Готові кадри мають опинитися в підтеці `spin/` поряд з `index.md` (наприклад, `src/content/igrashky/<slug>/spin/`). Кількість кадрів визначається автоматично.

4. **Перевірте.** Запустіть `npm run dev` — нова іграшка з’явиться в каталозі,
   а її сторінка буде доступна за адресою `/igrashky/<slug>/`.

## Як підготувати фотографії

Якщо ви створюєте іграшку вручну або хочете окремо підготувати та оптимізувати фотографії для вже існуючої іграшки, скористайтеся допоміжним скриптом:

```bash
node scripts/prepare-photos.mjs <шлях-до-оригіналів-чи-файлу> <тека-іграшки>
```

Приклад (тека іграшки має відповідати її slug):
```bash
node scripts/prepare-photos.mjs ~/foto/vedmedyk-tymko src/content/igrashky/vedmedyk-tymko
```

Скрипт:
1. Зчитує одне або декілька зображень із вказаного шляху (підтримуються `.jpg`, `.jpeg`, `.png`, `.webp`, `.heic` тощо).
2. Сортує їх за алфавітом.
3. Перше фото (або файл, що містить `cover` у назві) стає головним фото `cover.webp`. Решта стають додатковими фото `gallery-1.webp`, `gallery-2.webp` тощо.
4. Стискає зображення до макс. 2400px по довшій стороні (без збільшення дрібних зображень) та конвертує у формат WebP з якістю 90%.
5. Перед записом автоматично очищує старі версії фотографій (`cover.webp` та `gallery-*.webp`) у цільовій теці іграшки.

## Як підготувати кадри обертання

Оглядач обертання автоматично підхоплює кадри `frame-01.webp`, `frame-02.webp` … з підтеки `spin/` у теці іграшки. Щоб підготувати їх з набору фотографій або відеозапису обертання, скористайтеся допоміжним скриптом:

```bash
node scripts/prepare-spin.mjs <оригінали-чи-відео> [тека-призначення] [--frames N]
```

Параметри:
- `<оригінали-чи-відео>` — шлях до теки з оригіналами фотографій або шлях до відеофайлу обертання (`.mp4`, `.mov`).
- `[тека-призначення]` — опціональний шлях для збереження результатів. Якщо не вказано, використовується `src/content/igrashky/<назва-вхідного-файлу-чи-теки>/spin/`.
- `--frames N` — кількість кадрів, які необхідно витягти з відео (за замовчуванням `36`). Працює лише за умови вхідного відеофайлу.

> [!NOTE]
> Для роботи з відеофайлами на вашому комп'ютері мають бути встановлені утиліти `ffmpeg` та `ffprobe`, доступні в системному шляху змінних середовища (PATH).

Скрипт:
1. Якщо на вхід подано відео — витягує `N` кадрів за допомогою `ffmpeg` у тимчасову теку.
2. Читає всі зображення з теки (у порядку імен файлів).
3. Зменшує кожне до 1200 px по довшій стороні.
4. Конвертує у WebP якістю 80.
5. Перейменовує послідовно у `frame-01.webp`, `frame-02.webp`, …

> Порада: знімайте іграшку на поворотному столику з однакової відстані,
> 24–36 кадрів за повний оберт дають плавне обертання.

## Особливості кешування в Astro 5 (Content Layer)

Astro 5 кешує дані контент-колекцій у тимчасовій теці `.astro/`. Через це при видаленні файлів іграшок або значній зміні шляхів до фотографій ви можете зіткнутися з помилкою збірки (наприклад, `ImageNotFound` для файлів, які вже були видалені).

**Вирішення:**
Якщо dev-сервер видає помилку про відсутність старих картинок, просто зупиніть процес у терміналі (`Ctrl + C`) та запустіть його знову:

```bash
npm run dev
```

Сервер оновить кеш, і збірка пройде успішно.

## Розгортання на Cloudflare Pages

1. Завантажте проєкт у репозиторій GitHub (або GitLab).
2. У панелі Cloudflare відкрийте **Workers & Pages → Create → Pages**
   і під’єднайте репозиторій.
3. Укажіть параметри збірки:
   - **Framework preset:** Astro;
   - **Build command:** `npm run build`;
   - **Build output directory:** `dist`.
4. Натисніть **Save and Deploy**. Cloudflare збере сайт і надасть адресу
   виду `https://<проєкт>.pages.dev`.
5. Впишіть цю адресу в поле `site` у `astro.config.mjs` і зробіть новий коміт,
   щоб карта сайту та Open Graph використовували правильний домен.

Кожен наступний коміт у головну гілку автоматично оновлюватиме сайт.

## Структура проєкту

```
babusyni-igrashky/
├── astro.config.mjs           # налаштування Astro та карти сайту
├── src/
│   ├── config.ts              # назва сайту й контакти
│   ├── content.config.ts      # схема колекції іграшок
│   ├── content/igrashky/      # колекція іграшок
│   │   └── <slug>/            # самодостатня тека іграшки
│   │       ├── index.md       # опис та метадані іграшки
│   │       ├── cover.webp     # головне фото
│   │       ├── gallery-1.webp # додаткові фото
│   │       └── spin/          # кадри обертання (якщо є)
│   │           ├── frame-01.webp
│   │           └── ...
│   ├── assets/                # спільні графічні ресурси сайту
│   ├── components/            # компоненти сторінок
│   ├── layouts/               # базовий макет із SEO
│   ├── pages/                 # головна та сторінка іграшки
│   └── styles/global.css      # змінні палітри й базові стилі
├── public/
│   ├── favicon.svg
│   └── og-default.png
└── scripts/                   # допоміжні скрипти автоматизації
    ├── new-toy.mjs            # головна точка входу для створення іграшок (GUI або CLI)
    ├── pipeline.mjs           # загальний конвеєр обробки фото та формування маніфесту
    ├── prepare-photos.mjs     # обробка та оптимізація фотографій галереї
    ├── prepare-spin.mjs       # підготовка кадрів обертання (з картинок або відео)
    ├── server.mjs             # локальний HTTP-сервер для Web GUI
    └── gui/                   # файли інтерфейсу користувача (HTML, CSS)
```
